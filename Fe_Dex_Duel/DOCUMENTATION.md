## 1. Overview

**GameFi Prediction Arena** (DexDuel) is a fully on-chain, decentralized prediction market game running on the **OneChain** blockchain (a Sui VM-compatible network). The core promise is a **lossless model**:

- Players stake **OUSDT** (a testnet mock USDT token) and predict whether a crypto asset's price will go **UP** or **DOWN** within a tournament window.
- **Winners** share the **yield** generated by the prize pool.
- **Losers** always get **100% of their original stake back** — zero risk on principal.

All game state (sessions, predictions, scores, rewards) is stored and executed in **Move smart contracts** on OneChain testnet. The frontend is a pure read/write interface against those contracts.

`WalletProvider` is configured with `autoConnect: false` and `storage: null` — the wallet state is not persisted across browser sessions, users must reconnect every time.

#### AccountSelectionGate (`components/AccountSelectionGate.tsx`)

A full-screen overlay modal that fires **once, immediately after the wallet connects** when the wallet exposes multiple accounts.

**Logic:**
```
connectionStatus changes → "connected"
  ├── accounts.length <= 1  →  no modal, proceed silently
  └── accounts.length > 1   →  open modal asking user to pick one account
        ↓ user clicks an account
        switchAccount({ account: target })  →  close modal
```

On disconnect → modal resets.

```

  ├─ 2. Check: status === "live"?            → error if not
  ├─ 3. Check: OUSDT coin object exists?    → error if not ("Claim faucet first")
  ├─ 4. Check: balance >= entryFeeRaw?      → error if insufficient
  │
  ├─ 5. Read selectedDirection for this sessionId  (UP=1 or DOWN=2, default UP)
  │
  ├─ 6. setJoiningSessionId(sessionId)
  │
  ├─ 7. Call joinGame({ sessionId, roundId, registryId, direction, coinObjectId, entryFeeRaw })
  │       → buildJoinGameTx()  → sign via wallet  → execute on chain
  │
  ├─ 8. On success: save txDigest → show explorer link
  │
  └─ 9. Refetch balance + tournaments (parallel)
```

2. Builds + signs + submits `ousdt::claim_faucet(FAUCET)` on-chain.
3. On success shows "+100 USDT sent" badge.
4. On failure shows the error message below the button.

### 4.6 Leaderboard — `/leaderboard`

**File:** `app/leaderboard/page.tsx`  
**Hook:** `hooks/useLeaderboard.ts`

Displays a global player ranking built entirely from on-chain `ScoreUpdated` events.

#### How Scores Are Aggregated (`useLeaderboardRows`)

```
1. client.queryEvents({ MoveEventType: "leaderboard::ScoreUpdated", limit: 400, order: "descending" })

2. For each event, extract:
   - player (wallet address)
   - points_earned (>0 = win event, 0 = no-win update)
   - new_total_score (latest known total)
   - current_streak
   - timestampMs

3. Group by player (lowercased address):
   - totalScore = max new_total_score seen
   - currentStreak = max current_streak seen
   - winEvents = count of events where points_earned > 0
   - updateEvents = total event count for this player
   - lastUpdateMs = most recent

4. Sort: primary by totalScore desc, secondary by lastUpdateMs desc
5. Assign rank 1, 2, 3, …
```

Refreshes every **15 seconds**.

### 5.2 Transaction Execution Pattern

All write hooks (`useJoinGame`, `useFaucet`, `useClaimReward`, etc.) follow the same 4-step pattern, implemented via `lib/walletFeatures.ts → signTransactionForExecution()`:

```
Step 1: Build transaction
  buildXxxTx(params)  →  Transaction object (from @onelabs/sui/transactions)
  tx.setSender(account.address)

Step 2: Serialize to BCS bytes
  const bytes = await tx.build({ client })
  // Uses the testnet RPC client to resolve all object references

Step 3: Sign the raw bytes
  signTransactionForExecution(wallet, account, tx, bytes)
  // Tries wallet feature "sui:signTransactionBlock" first (passes raw Uint8Array)
  // Falls back to "sui:signTransaction" if the first feature is unavailable
  // Resolves the correct chain ID (prefers "one:testnet", then "onechain:testnet", then "sui:testnet")

Step 4: Execute on chain
  client.executeTransactionBlock({
    transactionBlock: transactionBlockBytes,
    signature,
    options: { showRawEffects: true },
  })
```

> **Why pre-build BCS?** OneWallet has a bug where its JSON→BCS re-encoding silently drops `TransferObjects` commands. Pre-building and signing the raw bytes bypasses this issue entirely.

## 8. API Routes

All routes live in `app/api/market/`. They proxy to [Finnhub](https://finnhub.io/) using a server-side API key. The key lookup order is:
```
process.env.FINNHUB_API_KEY  →  process.env.NEXT_PUBLIC_FINNHUB_API_KEY  →  hardcoded fallback key
```

All responses have `Cache-Control: no-store`.

### `GET /api/market/quote`

| Param | Required | Default | Description |
|-------|----------|---------|-------------|
| `symbol` | No | `BINANCE:BTCUSDT` | Finnhub symbol string |

**Upstream:** `https://finnhub.io/api/v1/quote?symbol={symbol}&token={key}`  
**Response:** `FinnhubQuote { c, d, dp, h, l, o, pc, t }`

| Field | Meaning |
|-------|---------|
| `c` | Current price |
| `d` | Change (absolute) |
| `dp` | Change (%) |
| `h` | 24h high |
| `l` | 24h low |
| `o` | Open price of current day |
| `pc` | Previous close price |
| `t` | Timestamp (unix seconds) |

### `GET /api/market/candles`

| Param | Required | Default | Description |
|-------|----------|---------|-------------|
| `symbol` | No | `BINANCE:BTCUSDT` | Finnhub symbol string |
| `resolution` | No | `5` | Candle resolution in minutes |
| `from` | No | `now − 6h` | Unix timestamp (seconds) |
| `to` | No | `now` | Unix timestamp (seconds) |

Validates `from < to`, returns 400 otherwise.  
**Upstream:** `https://finnhub.io/api/v1/crypto/candle`  
**Response:** `FinnhubCandles { c, h, l, o, s, t, v }`

### `GET /api/market/symbols`

| Param | Required | Default | Description |
|-------|----------|---------|-------------|
| `exchange` | No | `BINANCE` | Exchange name |
| `quote` | No | `USDT` | Quote currency filter |

**Upstream:** `https://finnhub.io/api/v1/crypto/symbol?exchange={exchange}`  
Filters the Finnhub response to only include symbols whose `.symbol` ends with the `quote` currency.  
**Response:** `FinnhubSymbol[] { description, displaySymbol, symbol }`

## 11. Game Mechanics

### Tournament Lifecycle

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Tournament Lifecycle                          │
│                                                                     │
│  [Admin: Create]   [Admin: Start]   [Users: Predict]  [Round Ends] │
│       ↓                  ↓                ↓                 ↓      │
│   UPCOMING     →       LIVE      →      LIVE      →      ENDED     │
│                                                                     │
│  isActive=false    isActive=true    predictions         isEnded=true│
│  no price lock     price locked      accepted           settlement  │
│                    (start_price)                         pending    │
│                                                                     │
│                                                         ↓           │
│                                                  [Users: Claim]    │
│                                                  Winners: stake +  │
│                                                    yield share     │
│                                                  Losers: stake     │
│                                                    returned only   │
└─────────────────────────────────────────────────────────────────────┘
```

### Status Computation (client-side)

```typescript
// hooks/useOnChainTournaments.ts
function statusFromTimes(nowMs, startTimeMs, endTimeMs, isActive, isEnded, isSettled) {
  if (isSettled || isEnded || (endTimeMs > 0 && nowMs >= endTimeMs))
    return "ended";
  if (isActive)
    return "live";
  if (startTimeMs > 0 && nowMs < startTimeMs)
    return "upcoming";
  return "upcoming";   // default
}
```

### Round Object Fields (on-chain, in `game_round::Round`)

| Field | Type | Description |
|-------|------|-------------|
| `round_id` | u64 | Round number |
| `coin_symbol` | vector\<u8\> | ASCII bytes of the crypto symbol (e.g. [66,84,67] = "BTC") |
| `start_time` | u64 | Tournament start time (ms) |
| `end_time` | u64 | Tournament end time (ms) |
| `entry_fee` | u64 | Entry fee in raw OUSDT units |
| `up_count` | u64 | Number of UP predictions |
| `down_count` | u64 | Number of DOWN predictions |
| `total_participants` | u64 | Total players joined |
| `total_pool` | Balance\<OUSDT\> | Total OUSDT staked |
| `yield_pool` | Balance\<OUSDT\> | Yield generated for winners |
| `winner_direction` | u8 | 0 = undecided, 1 = UP won, 2 = DOWN won |
| `status.is_active` | bool | True after `start_game()` is called |
| `status.is_ended` | bool | True after end_time passes |
| `status.is_settled` | bool | True after rewards are distributed |

### OUSDT Token

- **Testnet-only** mock USDT used as the game currency.
- **6 decimal places**: `1 OUSDT = 1,000,000 raw units`
- **Faucet**: anyone can claim **100 OUSDT** by calling `ousdt::claim_faucet(FAUCET_OBJECT)` — no cooldown.
- The token type is `{PACKAGE_ID}::ousdt::OUSDT`.

### Read Flow (how the frontend fetches data)

```
React Component mounts
  → custom hook (e.g. useOnChainTournaments)
      → useQuery (React Query)
          → queryFn runs:
              ┌─ chain reads via useSuiClient() ─────────────────┐
              │  client.queryTransactionBlocks(...)               │
              │    → find create_game_session txs                 │
              │    → extract object IDs from objectChanges        │
              │  client.multiGetObjects(roundIds)                 │
              │    → parse Round fields (coin_symbol, times, etc) │
              │  client.queryEvents(MoveEventType)                │
              │    → PlayerJoinedGame / ScoreUpdated events       │
              └───────────────────────────────────────────────────┘
          ← structured data returned
      ← cached in React Query (TTL & poll interval per hook)
  → component renders with fresh data
```

### Write Flow (how users sign and submit transactions)

```
User Action (e.g. click "Join + Pay 100 USDT")
  │
  ├─ Validation (client-side guards)
  │
  ├─ Build Transaction object
  │    lib/transactions.ts → buildJoinGameTx(...)
  │    tx.setSender(account.address)
  │
  ├─ Serialize to BCS bytes
  │    bytes = await tx.build({ client })
  │    (resolves all object references via testnet RPC)
  │
  ├─ Sign raw bytes
  │    lib/walletFeatures.ts → signTransactionForExecution(wallet, account, tx, bytes)
  │    Tries: sui:signTransactionBlock  ←― passes raw Uint8Array (avoids re-encoding bug)
  │    Falls back to: sui:signTransaction
  │    Resolves chain: "one:testnet" preferred
  │
  ├─ Execute on OneChain
  │    client.executeTransactionBlock({ transactionBlock, signature })
  │
  ├─ On success: save txDigest, show Explorer link
  │
  └─ Trigger data refetch (React Query invalidation)
```
